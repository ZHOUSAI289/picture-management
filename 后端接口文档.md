# 校园图库管理平台 - 后端接口文档

## 1. 文档说明

本文档描述了校园图库管理平台的后端 API 接口规范，用于指导前后端开发人员进行接口对接。

---

## 2. 基础信息

### 2.1 基础信息

| 环境 | URL |
| :--- | :--- |
| 开发环境 | `http://localhost:8080/api` |
| 测试环境 | `http://test-api.example.com/api` |
| 生产环境 | `http://api.example.com/api` |

### 2.2 协议要求

- 使用 **HTTPS** 协议
- 所有接口必须通过 **TLS 1.2** 及以上版本
- 请求和响应均使用 **UTF-8** 编码

### 2.3 数据格式

- 请求体和响应体均为 **JSON** 格式
- 响应 Content-Type：`application/json;charset=UTF-8`

### 2.4 状态码说明

| 状态码 | 含义 | 说明 |
| :--- | :--- | :--- |
| 200 | 成功 | 请求成功，返回数据 |
| 201 | 创建成功 | 资源创建成功 |
| 400 | 请求错误 | 请求参数有误 |
| 401 | 未授权 | 需要登录或 Token 无效 |
| 403 | 禁止访问 | 无权限访问该资源 |
| 404 | 资源不存在 | 请求的资源不存在 |
| 500 | 服务器错误 | 服务器内部错误 |

### 2.5 通用响应格式

**成功响应**：
```json
{
  "code": 1,
  "msg": "success",
  "data": {}
}
```

**错误响应**：
```json
{
  "code": 0,
  "msg": "错误描述信息",
  "data": null
}
```

## 3. 接口列表

### 3.1 图片管理

#### 3.1.1 获取图片列表

**功能描述**：获取图片列表，支持分页、搜索、筛选

**请求路径**：`/api/images`

**请求方式**：`GET`

**请求参数**（Query Parameters）：

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| page | Integer | 否 | 页码，默认 1 |
| pageSize | Integer | 否 | 每页数量，默认 10 |
| title | String | 否 | 标题关键词搜索 |
| category | String | 否 | 图库分类筛选 |
| startDate | String | 否 | 开始日期（格式：YYYY-MM-DD） |
| endDate | String | 否 | 结束日期（格式：YYYY-MM-DD） |
| status | String | 否 | 状态筛选（pending/approved/rejected/published） |

**成功响应**：

```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "title": "校园主楼",
        "category": "校园建筑",
        "photographer": "张三",
        "date": "2024-01-15",
        "url": "https://picsum.photos/400/400?random=1",
        "status": "published",
        "createdAt": "2024-01-15T10:30:00Z",
        "updatedAt": "2024-01-15T11:00:00Z"
      }
    ],
    "total": 100,
    "page": 1,
    "pageSize": 10,
    "totalPages": 10
  }
}
```

**响应字段说明**：

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | Integer | 图片ID |
| title | String | 图片标题 |
| category | String | 图库分类 |
| photographer | String | 拍摄者 |
| date | String | 拍摄日期 |
| url | String | 图片URL |
| status | String | 状态（pending/approved/rejected/published） |
| createdAt | String | 创建时间 |
| updatedAt | String | 更新时间 |

#### 3.1.3 上传图片

**功能描述**：上传图片并保存相关信息

**请求路径**：`/api/images/upload`

**请求方式**：`POST`

**请求格式**：`multipart/form-data`

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| title | String | 是 | 图片标题 |
| category | String | 是 | 图库分类 |
| enableCustomCategory | Boolean | 否 | 是否启用自定义分类（默认false） |
| customCategory | String | 否 | 自定义分类名称（当enableCustomCategory为true时必填） |
| photographer | String | 是 | 拍摄者姓名 |
| date | String | 是 | 拍摄日期（格式：YYYY-MM-DD） |
| image | File | 是 | 图片文件（支持 jpg, jpeg, png, gif, 大小限制10MB） |
| description | String | 是 | 图片简介 |

**成功响应**：

```json
{
  "code": 1,
  "msg": "上传成功",
  "data": {
    "id": 7,
    "title": "校园主楼",
    "category": "校园建筑",
    "photographer": "张三",
    "date": "2024-01-15",
    "url": "https://cdn.example.com/images/7.jpg",
    "description": "校园主楼正面全景图",
    "status": "pending",
    "createdAt": "2024-01-15T10:30:00Z"
  }
}
```

## 4. 数据模型

### 4.1 图片模型（Image）

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | Integer | 是 | 图片ID（自动生成） |
| title | String | 是 | 图片标题 |
| category | String | 是 | 图库分类 |
| photographer | String | 是 | 拍摄者 |
| date | String | 是 | 拍摄日期 |
| url | String | 是 | 图片存储URL |
| description | String | 是 | 图片简介 |
| status | String | 是 | 状态（pending/approved/rejected/published） |
| createdAt | String | 是 | 创建时间（自动生成） |
| updatedAt | String | 是 | 更新时间（自动生成） |

### 4.2 分类模型（Category）

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | Integer | 是 | 分类ID（自动生成） |
| name | String | 是 | 分类名称 |
| count | Integer | 是 | 该分类下的图片数量 |
| createdAt | String | 是 | 创建时间（自动生成） |

### 4.3 图片状态说明

| 状态值 | 说明 |
| :--- | :--- |
| pending | 待审核 |
| approved | 审核通过（未发布） |
| rejected | 审核拒绝 |
| published | 已发布（首页可见） |
